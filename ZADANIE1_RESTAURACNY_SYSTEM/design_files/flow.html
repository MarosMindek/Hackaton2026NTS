Module 04 · Restaurant · Journey map

One booking, three surfaces, five phases.

A single reservation flows across the guest's phone, the employee console and the kiosk — with the PMS, KDS and payment gateway exchanging state in the background. This is the canonical happy path; branches for external diners, packed lunches, room service and assisted dining are noted on the right.

Branch · Room service

Same flow, different stage.

Guest skips reservation & kiosk; order goes to in-room queue with delivery window. PMS auto-fills the room number; the runner sees a separate KDS rail.

G·02 → G·09 · skip kiosk
Branch · External diner

No PMS profile.

Walk-in flow asks for a card-on-file before pre-order. All-inclusive surfaces never appear. Receipts emailed; the rest of the flow is identical.

FORK at NFC · request card
Branch · Packed lunch / takeaway

Tomorrow, not tonight.

Guest schedules a pick-up time; KDS receives the order during the previous evening prep. No seating, no service — kiosk shows pickup queue.
