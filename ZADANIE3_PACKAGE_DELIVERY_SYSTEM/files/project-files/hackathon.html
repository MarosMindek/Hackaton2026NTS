01
⏱ 2 hodiny 🛠 Claude Design 👥 Teamová úloha

Zadanie úlohy

Každý team dostane jeden modul rozsiahleho informačného systému pre Kúpele Bardejov — jedno z najstarších a najvýznamnejších kúpeľných miest v strednej Európe. Systém bude slúžiť zamestnancom kúpeľov na každodenné riadenie prevádzky.

Vašou úlohou je navrhnúť UX/UI dizajn tohto modulu — wireframy, user flows a finálny vizuálny dizajn. Využite Claude Design na generovanie a iterovanie dizajnových riešení. Dizajn musí byť v súlade s FPT Digital Brand Guidelines — farebná paleta, typografia, komponenty a ikonografika — aby boli všetky moduly vizuálne kompatibilné a tvorili jednotný systém.

🐛 Úloha 2

Bug Hunt & Dokumentácia

Dostanete zdrojový kód neznámej aplikácie. Vašou misiou je nájsť čo najviac bugov a vytvoriť kvalitnú dokumentáciu.

02
⏱ 1,5 hodiny 🐍 Python 📝 Dokumentácia

Zadanie úlohy

Stiahnite si zdrojový kód mystery aplikácie (mystery_app.zip). Je to Python aplikácia — systém správy knižnice — no nie je bez chýb. Kód bol napísaný rýchlo a bez riadneho code review.

🎯 Čo robiť

  • Prejdite kód a identifikujte bugy
  • Klasifikujte každý bug (severity, typ)
  • Navrhnite opravu pre každý bug
  • Napíšte technickú dokumentáciu kódu
  • Vytvorte README.md pre projekt

📊 Typy bugov na hľadanie

  • Logické chyby — nesprávne výpočty, wrong conditions
  • Bezpečnostné riziká — slabé heslovanie, injection
  • Performance — zbytočné O(n²) operácie
  • Error handling — chýbajúce try/except, None checks
  • Python antipatterns — mutable defaults, atď.

Štruktúra odovzdania

# Bug Report Template
## Bug #N: [Názov bugu]
**Súbor:** app.py / models.py
**Riadok:** XX
**Severity:** Critical / High / Medium / Low
**Typ:** Logic / Security / Performance / Null Pointer / ...
**Popis:** Čo je zlé a prečo
**Reprodukcia:** Ako sa bug prejaví
**Navrhovaná oprava:**
```python
# opravený kód
```
⚠️ Hint: V kóde je viac ako 20 bugov rôznych typov. Tím, ktorý nájde a správne opíše najviac bugov s dobrou dokumentáciou, získa maximum bodov.
🚚 Úloha 3

Full Stack Delivery Aplikácia — Košice

Vytvorte aplikáciu na optimalizáciu doručovacích trás v Košiciach s rešpektovaním zón pre každé vozidlo.

03
⏱ 3,5 hodiny 🌐 Full Stack 📊 Optimalizácia 🗺 Košice

Zadanie úlohy

Spoločnosť KE-Delivery s.r.o. prevádzkuje doručovaciu flotilu v Košiciach. Každé vozidlo má priradeného vodiča a môže sa pohybovať iba v svojej mestskej časti. Vaša aplikácia musí navrhnúť optimálne doručovacie trasy pre všetky vozidlá na základe reálnych dát.

🗺 Zónový model: Košice sú rozdelené na 18 mestských častí. Každý vodič/vozidlo pokrýva práve jednu zónu. Zásilky sa musia priradiť vozidlu so správnou zónou a splnenie kapacitných obmedzení je povinné.

Vstupné dáta

📦 packages.csv — 1 050 zásilok
  • id, barcode
  • recipient_name
  • address, city_district
  • package_type, size
  • dimensions_cm, weight_kg
  • fragile, priority
  • payment_method, cod_amount_eur
  • insurance_value_eur
  • special_instructions, order_date
🚗 drivers.csv — 58 vodičov / vozidiel
  • driver_id, first_name, last_name
  • phone
  • vehicle_id, vehicle_make_model
  • vehicle_type, license_plate
  • max_weight_kg, max_volume_m3
  • max_packages_count
  • zone_mestska_cast ← kľúčové!
  • years_experience, shift_start, shift_end

Požiadavky na aplikáciu

  • Backend: API endpoint na načítanie zásilok a vodičov, algoritmus priraďovania zásilok podľa zóny + kapacity
  • Optimalizácia: Minimalizácia počtu zásilok, ktoré nie sú doručené (všetky vozidlá naplnené optimálne)
  • Frontend: Vizualizácia priradenia zásilok k vodičom, štatistiky vyťaženosti
  • Obmedzenia: weight_kg ≤ max_weight_kg, vozidlo iba vo svojej zóne, priority = Overnight má prednosť
💡 Bonus body za: Real-time mapa trasy (napr. Leaflet.js + OpenStreetMap), výpočet odhadovaného času doručenia, štatistický dashboard, export trasy pre vodiča ako PDF.

Ako vyberieme víťaza

Jednoduchý systém — každá úloha má víťaza, tím s najviac víťazstvami berie celkovú cenu. Pri remíze rozhoduje pondelková prezentácia.

1️⃣
Každá úloha = 1 víťaz
Porota vyberie najlepší výsledok pre každú z troch úloh
2️⃣
Skóre = počet víťazstiev
Max. 3 body. Team s najvyšším počtom vyhratých úloh vyhráva.
3️⃣
Remíza → pondelok
Ak majú viaceré teamy rovnaký počet víťazstiev, rozhoduje Elevation Peach prezentácia.
